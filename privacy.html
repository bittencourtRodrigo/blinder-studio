Privacy Policy

Last updated: 2026-05-16

Blinder Studio is a personal, single-user command-line tool that runs on the developer's own laptop. There is no hosted service, no signup, no shared database, and no telemetry. This policy explains exactly what data the Software touches, where it lives, and how to revoke access.

1. Who runs the Software

The Software is installed and operated by the developer (Rodrigo Bittencourt). No one else has access to the tokens, credentials, or content the Software handles. If you cloned the source and you are running your own copy, then you are the operator for that copy and this policy describes how the Software treats your data on your machine.

2. What data the Software accesses

The Software uses a single OAuth scope:

  • https://www.googleapis.com/auth/youtube.upload — the minimum permission needed to upload a video to a YouTube channel the authorized Google account owns.

With that scope, the Software:

  • Receives an OAuth refresh token from Google after the user grants consent.
  • Uses the refresh token to obtain short-lived access tokens that authorize videos.insert calls against the YouTube Data API v3.
  • Uploads video files produced locally on the operator's laptop along with metadata (title, description, tags, category, language, publishAt schedule time) supplied by the operator.

The Software does not read existing video data, channel analytics, comments, subscriber lists, or any personally identifiable information from YouTube. The youtube.upload scope is write-only by design.

3. Where data is stored

  • Refresh tokens are written to plain-text JSON files inside a credentials/ directory on the operator's local laptop. This directory is gitignored and is never transmitted off the machine by the Software.
  • Video files and metadata are written to a local output/ directory during production. They are uploaded directly from the laptop to YouTube via HTTPS.
  • SQLite database (blinder.db) on the laptop stores channel configurations, scheduled-upload manifests, mined topic ideas, and retry queue state. It never leaves the laptop.

No data is sent to any third party other than Google/YouTube, which receives the upload itself.

4. How data is used

The Software uses OAuth tokens solely to call the YouTube Data API on behalf of the operator's own Google account, for the express purpose of uploading and scheduling Shorts videos to YouTube channels the account owns. Data is not sold, shared, traded, advertised against, or used to train any machine-learning model. The Software complies with the Google API Services User Data Policy, including the Limited Use requirements.

5. Retention and deletion

  • Refresh tokens remain on the laptop until the operator deletes the corresponding file in credentials/ or revokes the grant via Google Account permissions.
  • Local video files and SQLite rows remain until the operator deletes them. The Software does not auto-purge.
  • There is no remote copy to delete — the Software has no server.

6. How to revoke access

To remove the Software's access to your Google account at any time, visit https://myaccount.google.com/permissions and remove “Blinder Studio”. Any refresh token previously issued becomes invalid immediately. You may additionally delete the credentials/ directory on your laptop.

7. Children

The Software is not directed at children and does not knowingly process data of children under 13. Operating the Software requires a Google account that meets Google's own age requirements.

8. Security

The Software stores tokens unencrypted on the operator's local disk, relying on the operating-system account that owns the laptop for access control. Operators should protect their machine accordingly (disk encryption, screen lock). Communication with Google's APIs uses HTTPS as provided by the official Google client libraries.

9. Third-party terms

By authorizing the Software, you also agree to the YouTube Terms of Service and the Google Privacy Policy, which govern how Google processes the OAuth grant and the API calls the Software makes on your behalf.

10. Changes to this policy

If this policy changes, the new version will be published at /privacy.html with an updated “Last updated” date. Material changes that affect data handling will be noted in the project's source repository commit history as well.
